# Olist Container (Batch_Data)

This container stores data for the batch processing pipeline.

-----------------------------------------------------------------------------------------------------------------------------------------------------------------
## Data Flow

- Source CSV files are uploaded to the Raw_Data folder in Azure Data Lake Storage (ADLS).
- Databricks Auto Loader reads the files using Structured Streaming (readStream).
- The ingested data is stored as Delta tables in the Bronze_Layer.
- Invalid records are written to the Quarantine tables for auditing and reprocessing.
- The Silver Layer performs schema validation, data type casting, duplicate removal, missing value handling, data standardization, and data quality checks.
- The cleaned data is transformed into business-ready datasets in the Gold_Layer using a Galaxy Schema (Fact Constellation) design.

-----------------------------------------------------------------------------------------------------------------------------------------------------------------
## Folders

- Raw_Data - Landing area for incoming source files.
- Bronze_Layer - Raw ingested data stored as Delta tables.
- Silver_Layer - Cleaned, validated, and standardized data.
- Gold_Layer - Business-ready fact and dimension tables for analytics and reporting.
- Quarantine - Records that fail validation or business rules.
- checkpoints__streams - Structured Streaming checkpoint data.
- checkpoints__schemas - Auto Loader schema metadata and schema evolution information.

## Note

***The Raw_Data folder serves as the shared source of incoming files for both the Batch_Data and Streaming_Data pipelines. The Gold Layer is generated by the same Gold notebook used by both pipelines, ensuring a consistent business-ready data model.***
